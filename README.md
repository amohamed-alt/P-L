# P-L Executive Dashboard

This site reads `data/dashboard-data.json`, which is updated by n8n from Google Sheets.

## Data flow

Google Sheets → n8n → GitHub `data/dashboard-data.json` → GitHub Pages `index.html`

## Required n8n GitHub node settings

- Resource: File
- Operation: Edit
- Owner: `amohamed-alt`
- Repository: `P-L`
- File path: `data/dashboard-data.json`
- File content: `{{ $json.fileContent }}`
- Commit message: `{{ 'Update dashboard data - ' + $now.toISO() }}`
- Branch: `main`

The file must already exist before using the Edit operation.

## Deploy

1. Upload all project files to the root of the `P-L` repository.
2. Keep the existing live `data/dashboard-data.json` generated by n8n. Do not overwrite it with the empty placeholder from this package.
3. Open GitHub repository Settings → Pages.
4. Select GitHub Actions as the source.
5. Push to `main`.

The included workflow deploys the repository whenever n8n or a user pushes a commit to `main`.

## Browser refresh

The dashboard uses a cache-busting query string and `cache: no-store`, so it requests the latest JSON when the page opens or the Refresh Data button is clicked.
